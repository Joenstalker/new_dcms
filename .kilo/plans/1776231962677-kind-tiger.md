# Plan: Handling GitHub Quality Gate Failures

## Summary

The error message "Cannot read image (this model does not support image input)" is **NOT a quality gate failure**. Based on analysis:

- Your push was **successful** - commits are on `main` and working tree is clean
- This error comes from an **AI feature** (likely Cursor IDE or GitHub Copilot vision) trying to analyze image files in your commit
- The actual quality gates (PHPStan, Pint, test coverage) must have passed

---

## What Quality Gates Do

The workflow (`.github/workflows/quality-gates.yml`) runs:

| Gate | Command | Failure Means |
|------|---------|---------------|
| Static analysis | `composer analyse` | PHPStan errors |
| Style check | `pint --test` | Code formatting issues |
| Coverage | `composer test:coverage` | Test coverage below threshold |

### On Push to `main`
- If any gate fails, the push is **rejected**
- Code does NOT enter the main branch
- You must fix locally and push again

### On Pull Request
- If any gate fails, PR **cannot be merged**
- Fix issues and push to update

---

## How to Check What Failed

### 1. Check GitHub Actions
1. Go to your repository on GitHub
2. Navigate to **Actions** tab
3. Find the failing workflow run
4. Check the specific step that failed

### 2. Run Locally First
Before pushing, run the same checks locally:

```bash
# Static analysis
composer analyse

# Style check  
vendor/bin/pint --test .

# Run tests
composer test
# or
php artisan test
```

### 3. Quick Fix Commands

```bash
# Fix code style automatically
vendor/bin/pint .

# Then commit and push again
git add -A && git commit -m "fix: code style" && git push
```

The "Cannot read image" error is from an AI vision feature, not CI/CD:

- **Likely source**: Cursor IDE's AI trying to preview an image in the UI, or GitHub Copilot analyzing images in a PR

---

## If Quality Gates Actually Fail

1. **Don't panic** - Your code is safe on your branch
2. **Run locally** - `composer analyse` and `vendor/bin pint .`
3. **Fix issues** - Address the PHPStan errors or style issues
4. **Commit and push** - `git add -A && git commit -m "fix: quality gate issues" && git push`

The quality gates protect your codebase - they're meant to catch issues before they reach production.
